# 実装バックログ（優先度順）

## P0（MVP開始直後）
1. 静的配信構成へ固定（常駐Node前提を廃止）
2. フロント初期化（静的エクスポート設定）
3. `job:fetch`（短時間終了）コマンドを実装
4. `sources` シード投入
5. URL 正規化ユーティリティ

## P1（MVP中核）
1. 複数ソース収集
2. 重複除去ロジック（canonical + title hash）
3. スコアリング処理
4. `public/data/trends.json` 生成
5. ホーム画面（注目 + 新着）

## P2（MVP完成）
1. カテゴリ画面
2. フィルタ（期間・ソース）
3. 監視用ログ出力
4. 単体テスト追加
5. cron 本番登録

## P3（運用安定化）
1. cron 実行時間の最適化（分割実行含む）
2. ログローテーション
3. バックアップ（JSON/設定）

## DoD（Definition of Done）
- 15分ごとの収集ジョブが失敗率 5% 未満で動作
- cron バッチが制限時間内に完了
- UI でカテゴリ切替と外部リンク遷移が動作
- HTTPS 配信で閲覧可能
