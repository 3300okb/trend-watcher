# AGENTS.md

このファイルは、エージェントが最初に読む単一の入口です。  
目的は **段階的開示 (Progressive Disclosure)** により、必要な時に必要な情報だけを読むことです。

## 0.5 最初の1手
- まず `.AGENTS_DOCS/INDEX.md` を開き、該当カテゴリを1つ選ぶ。
- 迷う場合は `PROJECT_OVERVIEW.md` だけ先に読む。

## 0. 基本方針
- まずこの `AGENTS.md` だけ読む。
- 次に、実行するタスクに必要なドキュメントだけを `.AGENTS_DOCS/` から追加で読む。
- 無関係なドキュメントは読まない。
- 不足情報が出た時だけ、追加で1ファイルずつ読む。

## 1. 読み進めルール
1. タスクを分類する（例: UI修正 / データ取得 / DB / デプロイ / ドキュメント）。
2. `.AGENTS_DOCS/INDEX.md` を開いて、該当カテゴリのファイル1つだけ読む。
3. まだ不足がある場合のみ、関連ファイルを追加で読む。
4. 実装後は必要な検証手順だけ実行し、結果を報告する。

## 2. 禁止事項
- タスクに関係ない `.AGENTS_DOCS/*` の一括読込。
- 必要性を説明せずに広範囲のファイルを同時に読む。
- 推測で設定や運用を変更する。

## 3. ドキュメント配置
- インデックス: `.AGENTS_DOCS/INDEX.md`
: どのドキュメントを最初に読むべきかを決める入口
- プロジェクト概要: `.AGENTS_DOCS/PROJECT_OVERVIEW.md`
: 構成と責務、変更対象の当たりを付けるための全体像
- 開発コマンド: `.AGENTS_DOCS/COMMANDS.md`
: タスクごとに実行する最小コマンドを判断する
- 実装規約: `.AGENTS_DOCS/CODING_RULES.md`
: 変更境界と検証方針を判断する
- データフロー: `.AGENTS_DOCS/DATA_FLOW.md`
: 入出力JSONと更新経路を確認する
- デプロイ運用: `.AGENTS_DOCS/DEPLOYMENT.md`
: 反映手順と反映後確認を行う

## 4. 更新ポリシー
- 新しい知見は、既存ファイルに追記する。
- 新規ファイル追加時は必ず `.AGENTS_DOCS/INDEX.md` にリンクを追加する。
- 長文化したら要約を先頭に置き、詳細は後半へ移す。
